## 0. 전제 — 데이터 기반 판단 근거

- 검색 유입의 **90% 이상이 `/orgs/기관명/` 상세 페이지로 직접 랜딩**됨 (홈 랜딩은 전체의 10% 수준) → 광고의 주전장은 메인 지도가 아니라 **상세 페이지 템플릿**
- 사용자의 **약 70%가 모바일**
- scroll 이벤트 발생률 약 58%, 평균 참여시간 약 52초 → 첫 화면 + 스크롤 1~2회 구간까지가 유효 노출 영역
- 검색어가 전부 "연봉/초봉" 중심 → 사용자가 원하는 정보(연봉)를 확인한 직후가 시선이 가장 확실히 지나가는 지점

이 문서의 슬롯 배치는 전부 위 근거에서 도출됨.

## 1. 상세 페이지 (`generate-org-pages.js`) — 우선순위 최상위

344개 기관 페이지 전체에 일괄 반영되므로 레버리지가 가장 크다.

### ① 연봉 정보 카드 바로 아래 — **최우선 슬롯**

- **위치**: `연봉 정보 (ALIO 공시 기준)` 카드가 끝나는 지점 ~ `주요 사업` 카드 시작 전 (line ~463 부근)
- **형식**: 반응형 인콘텐츠 광고 (Auto ads 또는 `<ins class="adsbygoogle">` 반응형)
- **근거**: 사용자가 페이지를 연 목적(연봉 확인)을 막 끝낸 시점 = 뷰어빌리티가 가장 높은 지점. 이탈 직전이 아니라 "정보는 얻었고 아직 스크롤할 의향이 남은" 타이밍이라 광고 회피가 상대적으로 낮음

### ② 전국 지사·지점 카드와 연봉 순위 카드 사이

- **위치**: `전국 본사·지점·사업소` 카드 뒤, `업종 연봉 순위` 카드 앞 (line ~520 부근)
- **형식**: 반응형 디스플레이 광고
- **근거**: 여기까지 스크롤한 사용자는 체류시간이 긴 고관여 사용자. 지사가 많은 대형 기관(한국농어촌공사·근로복지공단·한국산업인력공단 등 지사 30개↑) 페이지에서 특히 노출 기회가 많아짐

### ③ 모바일 앵커(하단 고정) 광고

- **위치**: `<head>`에 Auto ads 앵커 광고 허용 설정 (별도 슬롯 마크업 불필요, 애드센스 관리자 화면에서 자동광고 옵션으로 켜는 방식)
- **근거**: 모바일 70% 사이트에서 노출 100% 보장, 콘텐츠를 가리지 않아 UX 부담 낮음. 승인 직후 가장 손 안 대고 켤 수 있는 슬롯

## 2. 경영평가 랜딩페이지 (`generate-eval-page.js`)

### ④ 등급 그룹(S/A/B/C/D/E) 섹션 사이

- **위치**: 각 등급 섹션(`gradeSection()`) 사이 1개만
- **형식**: 인피드 광고
- **근거**: 네이버 색인 완료, 체류시간이 긴 페이지. 88개 기관을 등급별로 훑어보는 흐름을 방해하지 않는 지점에 1개만

## 3. 절대 붙이지 말아야 할 곳

| 위치 | 이유 |
|---|---|
| 메인 지도(`index.html`) 지도 영역 주변 | 홈 랜딩 비중이 전체의 10%뿐이라 수익 기여 미미. 지도 핀치줌/드래그 중 오클릭 유발 → **무효 클릭으로 계정 정지 리스크**. Kakao Maps SDK로 이미 무거운 페이지라 LCP 추가 악화도 우려 |
| 페이지 최상단(제목 카드 위) | 연봉 정보를 밀어내면 이탈률 상승. "콘텐츠보다 광고가 먼저"는 애드센스 심사에도 불리 |
| 오류신고 모달 주변 | 신고 접수(전환) 방해 |

데스크톱 사이드바 리스트 최하단에 1개 정도는 넣어도 되지만, 굳이 우선순위를 둘 필요는 없음 (모바일 비중이 훨씬 큼).

## 4. 실행 순서

1. **[대기 중]** 애드센스 승인 이메일 수신
2. 승인 즉시: `generate-org-pages.js`에 ①②번 슬롯 마크업 삽입 (`<ins class="adsbygoogle">` + `data-ad-slot` 코드) → 344개 페이지 일괄 재생성 → 커밋·푸시
3. 애드센스 관리자 화면에서 Auto ads → **앵커 광고**(③) 켜기 (코드 수정 불필요)
4. `generate-eval-page.js`에 ④번 슬롯 삽입 → `node generate-eval-page.js` 재생성 → 커밋·푸시
5. 승인 후 2주: 앵커 광고 ON/OFF를 GA 참여시간·이탈률로 A/B 비교
6. 병행 과제: CTR 낮은 고노출 키워드(예: "OO 연봉") 타이틀 개선 — 광고 수익의 최종 지렛대는 배치가 아니라 PV 자체이므로

## 6. 운영 시 주의사항

- **GitHub Pages 호스팅 정책**: "주된 목적이 상업적 운영"인 사이트를 금지하는 조항이 있음. 정보 사이트에 배너를 다는 수준은 회색지대이나, 수익이 본격화되면 **Cloudflare Pages**(무료, 광고 제한 없음)로 이전 검토. 도메인·구조 그대로 이전 가능해 SEO 영향 없음
- **성능**: 광고 스크립트는 반드시 lazy-load(뷰포트 진입 시 로드). 페이지당 슬롯 3개 초과 금지 (모바일 70% 사이트에서 LCP 저하는 검색순위에 직결)
- **무효 클릭 금지**: 본인/지인 클릭 절대 금지. 초기 계정은 특히 민감하므로 지도 UI처럼 오클릭 유발 배치도 금지
